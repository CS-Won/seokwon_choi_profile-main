Week 8 · Deutsch–Jozsa algorithm

Winter 2024 · Yonsei Premium Online Lecture — Lecturer: Seokwon Choi

Today we'll see the first algorithm where a quantum computer provably beats any classical one — Deutsch–Jozsa. It's a toy problem, but it's the cleanest illustration of how superposition + interference can be cashed in as a real speedup.

1. What "quantum advantage" means

To say a quantum computer is "better", you need a yardstick. The yardstick is time complexity: how many operations do you need as the input size grows? We track only the dominant term with Big-O notation:

quicksort on N items=O(NlogN)

2. The big complexity classes

  • P — problems a classical computer solves in polynomial time (e.g. sorting, multiplying matrices).
  • NP — problems whose solution can be verified in polynomial time (e.g. SAT, factoring). Whether P = NP is the most famous open problem in CS.
  • BQP — Bounded-error Quantum Polynomial time. Problems a quantum computer solves in polynomial time with error ≤1/3. This is the "efficiently solvable on a quantum computer" class.

Factoring is known to be in BQP (Shor's algorithm), but not yet known to be in P. That gap is what makes RSA suspicious in a quantum world.

3. Quantum parallelism — the trick

Take any boolean function f:{0,1}n→{0,1}. Define the quantum oracle

Uf​∣x⟩∣y⟩=∣x⟩∣y⊕f(x)⟩

where ⊕ is XOR. This is reversible (Uf​ is its own inverse), so it's a legal quantum gate.

Now prepare the input register in a uniform superposition over all x:

2n​1​x∑​∣x⟩∣0⟩Uf​​2n​1​x∑​∣x⟩∣f(x)⟩

We've just evaluated f on all 2n inputs at once. The catch (as always): measuring the second register collapses it to a single f(x). So you can't naively "read off all values". The art is to use interference to extract a global property of f.

4. Oracles and query complexity

We pretend Uf​ is a black box and count only how many times we call (= query) it. This is query complexity. The three flavors:

  • Deterministic: queries needed to always get the correct answer.
  • Probabilistic: queries needed if we allow a small error rate.
  • Quantum: queries needed using a quantum oracle.

5. Deutsch's algorithm (n = 1)

Suppose someone hands you a black box implementing a function f:{0,1}→{0,1}. There are four possible f. Two of them are constant (always output 0 or always 1), two are balanced (output 0 once and 1 once). Question: is f constant or balanced?

Classically, you'd query both inputs and compare: 2 queries. Deutsch's algorithm answers in 1 query.

The trick: phase kickback

Set up the ancilla in ∣−⟩=(∣0⟩−∣1⟩)/2​. Then

Uf​∣x⟩∣−⟩=(−1)f(x)∣x⟩∣−⟩

The XOR has been "kicked back" into a sign on the data register. Now Hadamard the data qubit before and after:

H∣0⟩∣−⟩=2​1​(∣0⟩+∣1⟩)∣−⟩
Uf​​2​1​((−1)f(0)∣0⟩+(−1)f(1)∣1⟩)∣−⟩
H​{±∣0⟩∣−⟩,±∣1⟩∣−⟩,​f constantf balanced​

Measure the data qubit. 0 means constant, 1 means balanced. One oracle call total. The relative phase put on ∣0⟩,∣1⟩ by f interferes constructively or destructively in the final Hadamard, depending on whether f is constant or balanced.

6. Deutsch–Jozsa — scaling it up

Same problem, but with f:{0,1}n→{0,1} promised to be either constant (same output for all inputs) or balanced (output 0 on half of inputs, 1 on the other half). Classical worst case: about 2n/2+1 queries. Quantum: 1 query.

The circuit looks just like Deutsch's with more wires:

  1. Initialize n data qubits at ∣0⟩ and 1 ancilla at ∣1⟩.
  2. Hadamard all n+1 qubits.
  3. Apply the oracle Uf​ once.
  4. Hadamard the n data qubits again.
  5. Measure the data register.

The result is ∣0⟩⊗n if and only if f is constant. Why? After the oracle, the data register holds amplitudes ∝∑x​(−1)f(x)∣x⟩. The final Hadamards perform an inner product with ∣0⟩⊗n; this is ±1 if f is constant and 0 if balanced.

7. Honest caveats

  • A randomized classical algorithm only needs O(1) queries to be right with high probability — so the speedup is huge against deterministic classical, but moot for randomized classical.
  • Real-world functions are rarely perfectly balanced or constant. The algorithm is mostly a stepping stone to Simon's algorithm and eventually to Shor's.

Recap

  • Quantum oracle Uf​∣x⟩∣y⟩=∣x⟩∣y⊕f(x)⟩ + phase kickback is the magic combo.
  • Deutsch–Jozsa: 1 quantum query vs up to 2n−1+1 deterministic classical queries.
  • Pattern reused later: Hadamard everywhere → oracle → Hadamard → measure.
