Week 4 · Quantum-inspired classical algorithm (Tang)

Winter 2024 · QISCA — Lecturer: Seokwon Choi

Ewin Tang showed that Kerenidis and Prakash's quantum recommendation algorithm has a classical counterpart with essentially the same complexity, up to polynomial factors, provided the input matrix is stored in a data structure that supports ℓ2-norm sampling queries (Tang, 2019). This is called dequantization: the apparent quantum speedup evaporates once we match input access models carefully. This week we walk through the construction step by step so you can see where the speedup actually went.

1. The input model — ℓ²-norm sampling oracle

We assume access to A∈Rm×n through a small set of cheap operations, all costing O(log(mn)):

  • Query: read Aij​.
  • Norms: return ∥Ai,⋅​∥ and ∥A∥F​.
  • Row sampling: sample a row index i with probability ∥Ai,⋅​∥2/∥A∥F2​.
  • In-row entry sampling: given i, sample column index j with probability Aij2​/∥Ai,⋅​∥2.

This is exactly what the BST in Kerenidis–Prakash gives the quantum algorithm — and the same BST also supports these classical queries. Tang's point: as soon as the classical algorithm has access to this oracle, the quantum speedup is mostly gone.

2. ℓ² sampling = classical analogue of quantum amplitudes

For any vector v∈Rn, the quantum state ∣v⟩=∑j​(vj​/∥v∥)∣j⟩ measured in the computational basis gives index j with probability

Pr[j]=∥v∥2vj2​​

— the same distribution you would get from length-squared sampling of v on a classical machine with the oracle above. Tang's slogan: length-squared sampling is the classical avatar of quantum measurement.

3. The goal

KP outputs a sample from a row Tk,i,⋅​ of a good rank-k approximation of the preference matrix. Tang aims to do exactly that classically: given the oracle above for A=T, output column index j with

Pr[j∣i]≈∥Tk,i,⋅​∥2Tk,ij2​​

in time O(poly(k)log(mn)) (Tang, 2019, Theorem 1).

4. Modified FKV — top singular subspace from a tiny sketch

The first move is essentially Frieze–Kannan–Vempala (Week 2). Sample r rows and then c columns of A using length-squared probabilities, getting a small r×c matrix W.

Let R∈Rr×n be the sampled, rescaled rows. By Week 2,

E[R⊤R]=A⊤A

and a matrix Bernstein bound gives ∥R⊤R−A⊤A∥F​≤η∥A∥F2​ with high probability whenever r=O(1/η2). Doing the same trick on columns, we end up with an r×c sketch W we can SVD in constant time (constant in m,n).

Lift the singular vectors of W back to an approximate right-singular basis of A:

v^ℓ​=σ^ℓ​1​R⊤w^ℓ​,ℓ=1,…,k

where (w^ℓ​,σ^ℓ​) are right singular vectors and values of W. With high probability the v^ℓ​ approximately span the top-k right singular subspace of A. We are still purely classical, and we never touched most of A — only the few sampled rows.

5. The new trick — sample from a projection without writing it

We want to sample from the projected row

T~i,⋅​=ℓ=1∑k​λℓ​v^ℓ​,λℓ​=⟨Ai,⋅​,v^ℓ​⟩

Writing out the full vector T~i,⋅​ would cost Ω(n) — unaffordable. Tang's key observation: we don't need to write it down, we only need to sample from it.

(a) Estimate the k coefficients λℓ

Each λℓ​ is an inner product between row i and the lifted singular vector v^ℓ​. Each v^ℓ​ is a short linear combination of the sampled rows (Step 4), so each λℓ​ reduces to a handful of inner products between row i and individual sampled rows. Each of those inner products can be estimated with length-squared sampling in time O(poly(k)/η2⋅log(mn)) — this is the classical "inner product estimator" trick (Tang, 2019, Lemma 3.5).

(b) Rejection sampling from the projected vector

Now we have explicit coefficients λ^1​,…,λ^k​ and the v^ℓ​ are stored as linear combinations of sampled rows (each of which we can length-squared sample). Tang then runs the following rejection sampler to draw an index j from T~i,⋅​:

  1. Pick ℓ∈{1,…,k} with probability λ^ℓ2​/∑ℓ′​λ^ℓ′2​ (a k-sided coin you can flip in O(k)).
  2. Length-squared sample an index j from v^ℓ​ using the oracle (recall v^ℓ​ is a combination of sampled rows, which support length-squared sampling for free).
  3. Accept j with probability(∑ℓ​λ^ℓ​(v^ℓ​)j​)2/(k∑ℓ​λ^ℓ2​(v^ℓ​)j2​). Otherwise go back to Step 1.

Why does this work? The probability that the sampler outputs j in one round is

Pr[output j]=ℓ∑​∑ℓ′​λ^ℓ′2​λ^ℓ2​​⋅1(v^ℓ​)j2​​⋅k∑ℓ​λ^ℓ2​(v^ℓ​)j2​(∑ℓ​λ^ℓ​(v^ℓ​)j​)2​∝(T~i,j​)2

which is exactly the length-squared distribution of the projected row. The acceptance probability bounds make sure the expected number of rounds is O(k); everything else is polylog(mn).

6. Runtime

Counting up the pieces:

TTang​=O(poly(k,1/ε,σmax​/σmin​)⋅log(mn))

Only polynomially slower than KP's quantum runtime O(poly(k)polylog(mn)). Crucially, the exponential gap with classical algorithms without sampling oracles (which scale polynomially in m,n) is preserved — Tang's algorithm beats those by an exponential factor too, just using the same input model as the quantum one.

7. So is HHL "useless"?

Reasonable question after seeing the dequantization. The honest answer from Tang (2019) is narrower: for this recommendation setup, the exponential-looking gap was an artifact of mismatched input models. HHL-style primitives may still matter for problems where:

  • State preparation and block encodings are cheap quantumly but the equivalent classical sampling distribution is provably hard (e.g. outputs from BQP circuits at large depth).
  • You need the full quantum state (not just samples) — e.g. to feed it into another quantum subroutine, instead of measuring at the end.

A complexity-theoretic warning sign: if every HHL-style problem could be fully simulated classically, we would have results like BQP⊆P, which almost nobody believes. Dequantization should be read as a careful boundary-marker, not as a death certificate for quantum linear algebra.

8. What to remember

  • Always separate algorithm runtime from input-model assumptions.
  • Length-squared sampling is the classical mirror of quantum amplitudes. With it, KP's recommendation pipeline can be reproduced in polylog time (up to polynomial slowdown).
  • Classical foundations: competitive guarantees (Drineas, Kerenidis, & Raghavan, 2002), fast Monte Carlo low rank (Frieze, Kannan, & Vempala, 2004), entrywise approximations (Achlioptas & McSherry, 2007). Tang (2019) shows how to assemble them with rejection sampling into a complete recommendation algorithm.

9. Sanity check — rank-1 case

Take A=uv⊤ with unit u,v and σ1​=1. Step 4 returns v^1​≈v. The single coefficient is λ^1​≈⟨Ai,⋅​,v⟩=ui​. The rejection sampler then draws j with probability proportional to vj2​ — exactly the same direction-of-v sample as the quantum algorithm in Week 3, and the natural ranking for the recommendation.
