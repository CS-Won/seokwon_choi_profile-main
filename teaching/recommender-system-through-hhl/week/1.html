Week 1 · HHL algorithm

Winter 2024 · QISCA — Lecturer: Seokwon Choi

The Harrow–Hassidim–Lloyd (HHL) algorithm is the backbone of this course. It solves certain linear systems Ax=b on a quantum computer in time that scales like polylogarithmic in the dimension, instead of polynomial in N classically — but only under strict assumptions and only for special kinds of outputs. We'll go through every step slowly so even someone seeing HHL for the first time can follow each line.

1. The problem in plain language

Imagine a square matrix A of size N×N and a vector b. The textbook task is to find x with Ax=b. The textbook answer is to invert: x=A−1b. For huge N (think N=230), even writing x down takes Ω(N) time. So HHL changes the rules: we ask only for a quantum state whose amplitudes encode x, and we read out a single expectation value like x†Mx at the end.

The standing assumptions we'll need:

  • A is Hermitian (A†=A). If it is not, replace it with the doubled-up (0A†​A0​) trick — same idea, slightly larger matrix.
  • A is sparse: at most s nonzero entries per row.
  • The condition number κ=λmax​/λmin​ is not too large. Eigenvalues lie in an interval like [1/κ,1].
  • We can prepare ∣b⟩, a normalized quantum state whose amplitudes are the entries of b.

2. Eigendecomposition: the magic lens

Because A is Hermitian, the spectral theorem gives us an orthonormal eigenbasis {∣uj​⟩}j=1N​ with real eigenvalues λj​:

A=j=1∑N​λj​∣uj​⟩⟨uj​∣,A∣uj​⟩=λj​∣uj​⟩

Expand ∣b⟩ in this eigenbasis (any orthonormal basis covers the whole space, so this is always possible):

∣b⟩=j=1∑N​βj​∣uj​⟩,βj​=⟨uj​∣b⟩

Why is this great? Because A−1 has the same eigenvectors and inverse eigenvalues:

A−1=j∑​λj​1​∣uj​⟩⟨uj​∣

So our target state is, up to normalization,

∣x⟩∝A−1∣b⟩=j∑​λj​βj​​∣uj​⟩

The whole algorithm is just: somehow attach the factor 1/λj​ to each amplitude βj​. The trick is doing this without knowing the λj​ in advance.

3. Quantum phase estimation — "label each eigenvector with its eigenvalue"

Phase estimation is a black-box quantum routine. Hand it a unitary U and an eigenvector ∣u⟩ with U∣u⟩=e2πiφ∣u⟩; it gives you back a register holding φ.

We pick U=eiAt0​ for some chosen t0​. Since A∣uj​⟩=λj​∣uj​⟩:

eiAt0​∣uj​⟩=eiλj​t0​∣uj​⟩

So the eigenvalue λj​ shows up as a phase. Choose t0​ so the phase λj​t0​/2π stays in [0,1), and phase estimation pulls λj​ into an ancilla register ∣λ~j​⟩. Applying it to ∣b⟩:

∣b⟩∣0⟩QPE​j∑​βj​∣uj​⟩∣λ~j​⟩

We don't actually know the eigenvalues — they are stored as amplitudes/labels, and the rest of the algorithm operates on those labels in superposition. Hamiltonian simulation of eiAt is the bottleneck here; modern block-encoding methods do it in time O~(s∥A∥t).

4. Controlled rotation — turn λ~j​ into 1/λ~j​

Add a fresh ancilla qubit and rotate it by an angle depending on the stored eigenvalue:

∣λ~j​⟩∣0⟩anc​↦∣λ~j​⟩(1−λ~j2​C2​​∣0⟩+λ~j​C​∣1⟩)

Here C is a constant chosen small enough that C/λ~j​≤1 for every possible λ~j​ — concretely C=O(1/κ) works, because the smallest λ~j​ is roughly 1/κ. This is a controlled Ry​ rotation built out of arithmetic on the eigenvalue register.

The full state after this step is

j∑​βj​∣uj​⟩∣λ~j​⟩(1−λ~j2​C2​​∣0⟩+λ~j​C​∣1⟩)

5. Uncompute and post-select

Run phase estimation backwards to uncompute the eigenvalue register (this puts ∣λ~j​⟩ back to ∣0⟩):

j∑​βj​∣uj​⟩∣0⟩(1−λ~j2​C2​​∣0⟩+λ~j​C​∣1⟩)

Now measure the last ancilla. Conditioned on the outcome ∣1⟩, the remaining state on the main register is

P1​​1​j∑​λ~j​Cβj​​∣uj​⟩∝j∑​λj​βj​​∣uj​⟩=A−1∣b⟩

That is exactly the (normalized) solution state we wanted.

The probability of getting the success outcome is

P1​=j∑​∣βj​∣2λ~j2​C2​=Θ(1/κ2)

because the worst eigenvalues are ∼1/κ. Using amplitude amplification, this can be boosted to O(1) in time O(κ), which is where the κ factor in the runtime ultimately comes from.

6. Putting the pieces together

Total runtime (Harrow, Hassidim, & Lloyd, 2009):

THHL​=O~(εlog(N)s2κ2​)

with later improvements bringing the κ dependence to nearly linear and the s dependence to almost linear. The headline is the logN: exponentially smaller than the N a classical solver pays. The caveats are exactly the assumptions: we never write x classically, we need state preparation for ∣b⟩, and we pay inκ,1/ε.

7. Mini sanity check — 2×2 example

Take A=(3/41/4​1/43/4​) and ∣b⟩=(∣0⟩+∣1⟩)/2​. Eigenpairs:

λ+​=1,∣u+​⟩=2​1​(∣0⟩+∣1⟩);λ−​=21​,∣u−​⟩=2​1​(∣0⟩−∣1⟩)

Then β+​=1,β−​=0, so A−1∣b⟩=β+​/λ+​∣u+​⟩=∣u+​⟩. The state is unchanged because ∣b⟩ happened to already be an eigenvector with eigenvalue 1. A nontrivial choice like ∣b⟩=∣0⟩ would give β+​=β−​=1/2​ and A−1∣0⟩∝11​∣u+​⟩+1/21​∣u−​⟩, i.e. the small-eigenvalue component is amplified — which is exactly the kind of behavior the controlled rotation step implements coherently.
