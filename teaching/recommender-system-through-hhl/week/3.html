Week 3 · Quantum recommendation (Kerenidis–Prakash)

Winter 2024 · QISCA — Lecturer: Seokwon Choi

Kerenidis and Prakash (2016) gave the first quantum recommendation algorithm whose runtime scales as poly(k)polylog(mn) for an m×n preference matrix T with a good rank-k approximation. The punchline: you can sample a good recommendation for any user without ever reconstructing the full approximate matrix. We'll go through each ingredient step by step.

1. The setup

Let T∈Rm×n be the (unknown) preference matrix, and T∈Rm×n the matrix of known entries (zeros where unknown). The modeling assumption is the usual low-rank one:

∥T−Tk​∥F​≤ε∥T∥F​

for some small constant rank k and tolerance ε. A "good recommendation" for user i is an index j where Tk,ij​ is large.

2. The BST data structure — fast state preparation

The first ingredient is a clever classical data structure. For each row i of T, store a binary search tree (BST) whose leaves hold the entries Tij2​ and whose internal nodes hold partial sums. Then maintain one more tree over the row norms ∥Ti,⋅​∥2.

With this structure (Kerenidis & Prakash, 2016, Section 3):

  • Reading or updating an entry costs O(log2(mn)).
  • A quantum algorithm can prepare any row state ∣Ti,⋅​⟩=∥Ti,⋅​∥1​∑j​Tij​∣j⟩ in O(polylog(mn)) time.

The amplitudes are Tij​/∥Ti,⋅​∥: measuring this state gives column index j with probability Tij2​/∥Ti,⋅​∥2 — exactly an ℓ2-sample of row i. This is the same statistic FKV used classically (Frieze, Kannan, & Vempala, 2004).

3. Quantum singular value estimation (QSVE)

Now we want phases that depend on singular values σj​ of T. Kerenidis and Prakash build a unitary block-encoding of T/∥T∥F​ from the BST and then apply phase estimation to it.

Write the SVD T=∑j​σj​uj​vj⊤​. For any input state ∣x⟩=∑j​αj​∣vj​⟩ expanded in right singular vectors, QSVE produces

∣x⟩∣0⟩QSVE​j∑​αj​∣vj​⟩∣σ~j​⟩

i.e. it tags each singular component with an estimate σ~j​ of σj​, with precision δ in time O(polylog(mn)/δ). This is the singular value version of the HHL phase-estimation step from Week 1.

4. Projecting onto the "large singular value" subspace

Once we have the singular value labels in superposition, we can keep only those above a threshold θ. Apply a controlled rotation that marks an ancilla as 1 iff σ~j​≥θ, uncompute the singular value register, and post-select on the marker. The result:

∣x⟩project​∣x≥θ​⟩:=∥Π≥θ​x∥1​j:σj​≥θ∑​αj​∣vj​⟩

In words: this is the orthogonal projection of x onto the span of right singular vectors whose singular values exceed θ — i.e. an approximate top-k subspace, since by Eckart–Young the large singular values carry essentially all the "signal" about T. Threshold choice θ=ησ for a parameter η∈(0,1) and σ=∥T∥F​/k​ is what Kerenidis and Prakash use to recover a rank-k projection (Kerenidis & Prakash, 2016, Theorem 3.2).

The connection to HHL is now visible: HHL applied controlled rotations of the form 1/λj​. KP applies a step-function rotation in σj​. The mechanics are the same — phase estimation, eigenvalue-conditioned rotation, uncompute, post-select.

5. From projection to recommendation

Pick the row i of the user we want to serve. Prepare ∣Ti,⋅​⟩ using the BST, project it onto the top singular subspace by the QSVE step above, and call the result ∣T~i,⋅​⟩. KP show that with high probability this state is ε-close to a row of a good rank-k approximation of T:

∥∣T~i,⋅​⟩−∣Tk,i,⋅​⟩∥≤ε

Now measure the column register in the computational basis. You get a column index j with probability

Pr[j∣i]=∣⟨j∣T~i,⋅​⟩∣2≈∥Tk,i,⋅​∥2Tk,ij2​​

i.e. a column proportional to its (squared) predicted utility — exactly a high-quality recommendation under the low-rank model. We never wrote the full matrix; we only sampled from its amplitudes.

6. Putting it together — runtime

Each step costs polylog in m,n and polynomial in k,1/ε:

TKP​(recommend for user i)=O(poly(k)polylog(mn))

Compare with classical reconstruction-based methods that cost poly(m,n). Apparent exponential speedup. The fine print is exactly the assumption of fast quantum state preparation via the BST — and that is where Tang's "dequantization" will strike next week.

7. Small sanity check

Consider a tiny example: a rank-1 matrix T=uv⊤ with unit vectors u∈Rm, v∈Rn and one singular value σ1​=1. The state we prepare for user i is ∣Ti,⋅​⟩=∑j​vj​∣j⟩ (up to sign of ui​). Measuring gives column j with probability vj2​ — that is, the "direction" of v drives the sample, and is exactly the ranking we want. Everything scales to higher rank by superposing across the singular vectors that pass the threshold.
