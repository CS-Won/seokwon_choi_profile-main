Week 4 · Tensor network and entanglement entropy

Spring 2025 · QISCA — Lecturer: Seokwon Choi

Primary Materials

[Slide][Material]

This week we want to measure quantum entanglement directly. Entanglement entropy tells us how fast information spreads after a quench — basically the cleanest single-number summary of "how quantum" the dynamics is. On a real noisy device this turns out to be tricky, so we lean on two ingredients: a classical baseline from tensor networks (MPS), and a measurement protocol called randomized measurement (RM) on the quantum side.

1. Defining entanglement entropy

Take a pure state ∣ψ⟩ on N qubits, density matrix ρ=∣ψ⟩⟨ψ∣. Split the chain into a subsystem A with L qubits and its complement with M=N−L qubits. The reduced density matrix of A is:

ρL​=TrM​ρ

The Rényi entropy of order n is:

SL(n)​=1−n1​log[Tr(ρLn​)]
(7)

As n→1, this turns into the more familiar von Neumann entropy:

SLvN​=−Tr(ρL​logρL​)

We focus on n=2. Why? Because Tr(ρL2​) is the purity of the subsystem, and there's a known way to estimate it from random measurements on a real quantum device.

2. Why this is hard on real hardware

The most direct way to get ρL​ is quantum state tomography. The problem: the number of measurements needed grows exponentially with the number of qubits. So tomography is out for anything bigger than maybe 10 qubits.

A nicer trick is SWAP-based many-body interference (SWAP-MBI): make two copies of the state, add one ancilla qubit, and run a swap test. The catch is connectivity — the ancilla has to talk to every other qubit in the subsystem. On superconducting hardware where each qubit only neighbors a few others, this gets impractical fast.

3. The randomized measurement (RM) protocol

The clean alternative is RM. The idea is simple in words: apply a random single-qubit unitary to each qubit in the subsystem, then measure in the computational basis. Repeat with many independent random unitaries, and combine the statistics in a specific way to extract Tr(ρA2​).

One round of RM looks like this:

  1. Prepare the state ρ with our Trotter circuit.
  2. For each qubit i∈A, sample a single-qubit unitary Ui(2)​ from the circular unitary ensemble (CUE), and apply it.
  3. Measure all qubits in A in the Z basis. Build up the bitstring distribution PU^a​​(jL​).
  4. Repeat with NU​ different random unitaries.

4. The purity formula

For one fixed random unitary instance U^a​, compute:

Xa​=2LjL​,jL′​∑​(−2)−D[jL​,jL′​]PU^a​​(jL​)PU^a​​(jL′​)

where D[jL​,jL′​] is the Hamming distance — the number of bits that differ between the two outcome strings:

D[jL​,jL′​]≡#{i∈A∣si​=si′​}

Average over the NU​ random unitaries:

X=NU​1​a=1∑NU​​Xa​NU​→∞​Tr(ρA2​)
(8)

And the Rényi-2 entropy comes from a final log:

S(2)=−logX

Intuitively: bitstrings that are nearly identical contribute positively to Xa​, while strings that look very different cancel. This Hamming-distance-weighted correlation is exactly what you would compute if you had two virtual copies of the state and ran a SWAP test on them — it's the same quantity by a different route.

5. The classical baseline (MPS-TDVP)

We still need a "ground truth" to compare against. Direct simulation works for N≤20 or so. But just a 60-qubit state in double precision would need about ∼16 exabytes of memory, which is, you know, not realistic.

So we use matrix product states (MPS): write the wavefunction as a chain of small tensors connected by bonds of dimension χ. Time evolution is then done with the time-dependent variational principle (TDVP), which handles both short- and long-range interactions. We use δt=0.1, χmax​=1000, and a truncation error of 10−12. That handles up to N=104 comfortably.

6. What we actually measured (at t=5.0)

On the device, the RM run uses NU​=60 random unitaries with TREX+DD+PT+ZNE on top — 1,800 circuit instances total, packed into 900 parallel pairs using Quantum Multi-Programming (QMP). The Rényi-2 entropy at t=5.0 comes out as:

  • Direct (QuSpin): S(2)≈2.07465
  • Qiskit statevector: ≈2.11936 (2.16% off Direct)
  • Qiskit RM protocol: ≈2.04836 (3.35% off statevector)
  • ibm_marrakesh: ≈2.08378 (1.73% off Qiskit RM)

So the entropy measured on a real, noisy quantum machine agrees with the noise-free classical simulation within a couple of percent. That's a decent result for a deep many-body circuit on hardware.

Quick note: SM doesn't work for this measurement. The RM protocol ends with random SU(2) unitaries, which means there's no easy "ideal answer" for the test circuit. So here we only use ZNE.

Recap

  • Rényi entropy: SL(n)​=1−n1​logTr(ρLn​)
  • Tomography and SWAP-MBI don't scale, so we use the RM protocol instead.
  • Key formula: X=2L∑(−2)−DPP′→Tr(ρA2​), then S(2)=−logX.
  • MPS-TDVP is our classical baseline for the bigger systems.
